Express 应用程序生成器

这里用的是express框架，
通过应用生成器工具 express-generator 可以快速创建一个应用的骨架。
express-generator 包含了 express 命令行工具。通过如下命令即可安装：
npm install express-generator -g

启动开发者工具

  1. 正常的启动的方式为根据package.json执行脚本语法
  2. npm start就能启动node服务
  3. 前端项目配置环境跨域访问node服务（下篇文章讲解）
  4. 这样启动无法对node的代码进行断点调试。

  5. 在node启动的-inspect的调试工具里自带热替换功能

    一：第一种。我用的是webstrom也就是在编辑器内断点调试（不过感觉很不爽，毕竟前端开发人员，还是习惯在浏览器的开发工具下断点调试，而且感觉也特慢）
    启动run里面有个debug模式。启动就OK，就可以在webstrom里面断点

    二：node –inspect bin/www
    上面代码中，–inspect参数是启动调试模式必需的。这时，打开浏览器访问http://127.0.0.1:3000就可以看到Welcome to Express说明node服务启动成功了
    接下来，就要开始调试了。一共有两种打开调试工具的方法，

    1）第一种是在 Chrome 浏览器的地址栏，键入 chrome://inspect或者about:inspect，
    会看到Remote Target下面有个Target 点击下面就会弹出node的调试工具

    2）第二种进入调试工具的方法，是在 http://127.0.0.1:3000 的窗口打开”开发者工具”，顶部左上角有一个 Node 的绿色标志，点击就可以进入（也可以在你自己启动的vue项目打开开发工具也会看到node标志）
