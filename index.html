安装codePush CLi

管理codepush账号需要通过NodeJS-based CLi
npm install -g code-push-cli 安装
code-push -v 查看版本
ps:安装nodejs先

创建一个codePush账号

在终端输入code-push register 会打开注册页面让你选择选择授权账号
授权通过之后，codepush会高速你access key，复制此key到终端即可完成注册
be9fea6a451d037a47c8cd1dbeea882e08085909

登陆成功后
Successfully logged-in. Your session file was written to C:\Users\DELL\AppData\Local\.code-push.config. You can run the code-push logout command at any time to delete this file and terminate your session.

在codepush服务器注册app

为了让codepush服务器知道你的app，我们需要向它注册app：在终端输入code-push app add <appName> <os> <platform>即可完成注册
注册完成之后会返回一套deployment key，key分生成环境和开发环境。 该key在后后面的步骤中会用到
如果在应用分为android和ios版，那么在向codepush注册应用的时候需要注册两个App获取两套deployment key
例如：code-push app add Myapp-Android android react-native

PS:

  • code-push app add 在账号里面添加一个新的app
  • code-push app remove 或者rm 在账号里移除一个app
  • code-push app rename 重命名一个存在app
  • code-push app list或者ls 列出账号下面的所有app 、
  • code-push deployment -k ls 获取 部署秘钥

提示如果在react-native link 的时候没有输入key 收入插入文件位置
android/app/src/main/res/values/strings.xml

ios/Github_RN/Info.plist

集成

  1. 第一步：在项目中安装react-native-code-push插件，终端进入你的项目根目录然后运行 npm install -s react-native-code-push
  2. 第二步：在android project中安装插件。codepush提供两种方式：rnpm和manual，下面用rnmp：运行npm i -g rnpm,来安装RNPM
  3. 第三步：运行rnpm link react-native-code-push。这条命令将会自动在android文件中添加好设置

发布更新

code-push支持两种发布更新的方式。

第一种方式通过code-push release-react发布更新

这种方式将打包与发布两个命令合二为一

命令格式：
code-push release-react <appNamen> <platform>

eg:
code-push release-react MyApp-ios ios
code-push release-react MyApp-Android android

高级版：    

code-push release-react MyApp-iOS ios --t 1.0.0 --dev false --d Production --des "1.优化操作流程" --m true

起中参数-t为二进制安装包的版本(android/app/build.gradle-versionName);-dev为是否启用开发和模式(默认为false);-d是要发布更新的环境为Production与Staging(默认为Staging);-des为更新说明；-m 强制更新

另外，我们可以通过code-push deployment ls <appName>来查看发布详情与此次更新的安装情况

第二方式：通过code-push release发布更新生成bundle

第一步：在工程目录里面新增bundles文件
第二步:运行命令打包 react-native bundle --platform 平台 --entry-file 启动文件 --bundle-output 打包js输出文件 --assets-dest 资源输出目录 --dev 是否调试
eg：    
ios：
react-native bundle --platform ios --entry-file index.js  --bundle-output  ./bundles/main.jsbundle --assets-dest  ./bundles --dev false
Android:
react-native bundle --platform android --entry-file index.js  --bundle-output  ./bundles/index.android.bundle --assets-dest  ./bundles --dev false

打包bundle结束后，就可以通过codepush发布更新了。在终端输入

code-push release <应用名称> <Bundles所有在目录> <对应的应用版本> --deploymentName:更新环境 --description: 更新描述 --mandatory：是否强制更新
以ios为例：
code-push release hotcode ./bundles/main.jsbundle 1.0.5 --deploymentName Staging  --description: '1.0.5描述' --mandatory：true

API_List类型

  • List.isList()判断是否是List类型

 List[1,2,3,4]

  • size获取List的长度

  • set方法用来设定指定下标的值set(下标，值)
    -1代表从后面往前面

    L3=L.set(0,'122')

  • delete 删除指定下标的值 delete(下标)

  • insert 差入值 insert(下标，值)

  • updata用来更新指定下标的值

    const  a=List([1,2,3,4])
const  b=a.updata(1,x=>x+100)
涉及业务，对下标为1的数据 进行加100

  • clear 清空并返回一个长度为0的新数组

    const c1=List([1,2,3,4])
const c2=C1.clear();
c1还是存在不变，c2为空了

  • push pop unshit shift同数组的同名方法

const set2=set1.setSize(2)
截取出来List[1,2]
